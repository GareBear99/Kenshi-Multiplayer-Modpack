# Contributing

## Before opening a pull request

Please keep changes focused and easy to review.

Good contribution types:
- documentation improvements
- compatibility notes
- load order corrections
- manifest updates
- tested mod safety notes
- installation clarifications

## For mod-related changes

Please include:
- exact mod name
- source link if available
- version information if available
- why it should be included, removed, or reclassified
- known conflicts
- multiplayer test notes

## Pull request standard

A good PR should explain:
- what changed
- why it changed
- what was tested
- what still needs validation
